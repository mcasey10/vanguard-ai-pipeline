1
Problem framing + competitive research
Defined user problem, target segment, and the gap in Vanguard's existing portal. Parallel competitive framing run in Claude.ai, ChatGPT, and Gemini to surface differing framings and reconcile into a refined problem statement.
T1
Claude.aiPrimary tool for problem framing and problem statement drafting. Used in the dedicated PM thread with full PRD context maintained throughout. ChatGPTRun in parallel with the same problem framing prompt to surface alternative framings. Key finding: ChatGPT produced 17 distinct issues vs. Claude.ai's more integrated analysis. Both inputs were reconciled into the final problem statement. GeminiThird parallel tool in the reconciliation experiment. Gemini's framing surfaced the "empathy gap" concept — users' emotional experience of the decision — which was underweighted in the initial Claude.ai draft. PerplexityConsidered for competitive landscape research — skipped because Claude.ai web search covered the same need without adding a separate tool to the workflow.

Multi-tool comparison experiment: same prompt run cold in ChatGPT and Gemini — Claude.ai version retained as primary after gap analysis. Gemini surfaced the Newly Self-Directed confirmation screen insight incorporated as REQ-EC-002 amendment.

6
Logic validation prototype
Bolt and Lovable evaluated side-by-side using identical prompts and the canonical sample dataset. Lovable produced correct output in one correction prompt — lot count accurate (32), both optimization modes working, all harvestable loss alerts firing, Wait & Save correct, FIFO vs specific lot ID contrast validated. Key finding: neither tool inferred account withdrawal priority order (taxable → IRA → Roth) without explicit instruction.
T3
LovablePrimary logic validation tool. Produced correct output in one correction prompt: 32 lots accurate, both Tax-First and Balance-First optimization modes working from the first prompt, all harvestable loss alerts firing, Wait & Save correct, FIFO vs specific lot ID contrast validated, Case 8 large withdrawal notice working. Selected as the reference implementation. Bolt.newEvaluated in parallel with Lovable using identical prompts. Bolt produced wrong lot count (43 vs 32 correct), missing optimization modes, and wrong account priority order (Roth IRA selected first). Bolt's free token allocation was exhausted before corrections could be applied. Evaluated but not selected as the reference implementation. v0 by VercelStrong candidate for React component generation — not evaluated at this stage. Better suited for the PD phase (translating Figma components to code) than for data-logic validation prototypes.
